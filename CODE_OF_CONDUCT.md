# Code of Conduct

## Our Pledge

We want `codex-bridge` to be a project where people can collaborate constructively, ask basic questions safely, and work through operational or technical problems without hostility.

## Expected Behavior

Examples of behavior that help this project:

- being respectful and patient
- assuming good intent while staying honest
- giving specific, actionable feedback
- welcoming questions from contributors with different experience levels
- focusing criticism on ideas, code, and process rather than people

## Unacceptable Behavior

Examples of behavior that are not acceptable:

- harassment or intimidation
- personal attacks
- hate speech
- sustained disruption of discussions
- publishing private information without permission
- deliberately hostile, demeaning, or exclusionary conduct

## Maintainer Responsibilities

Maintainers are expected to:

- clarify project standards
- respond to reports fairly and promptly
- protect contributors from abusive conduct
- apply standards consistently

## Scope

This code of conduct applies to:

- repository issues
- pull requests
- discussions related to the project
- direct project communication spaces when representing the project

## Reporting

- Email: [bakerthanhtung@gmail.com](mailto:bakerthanhtung@gmail.com)

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, or other contributions that do not align with this code of conduct.
